# src/art_style_search/ — pipeline conventions

Package-specific conventions for the top-level modules. Sub-packages have their own CLAUDE.md:
- `prompt/CLAUDE.md` — reasoning-model interface
- `workflow/CLAUDE.md` — orchestration
- `reporting/CLAUDE.md` — HTML report

## caption & generation

- `caption.py`: `CAPTION_SYSTEM` is the slim captioner contract. It enforces: (1) **quality standards** — precise color names, medium-appropriate terminology, spatial quantification, one observation per sentence, prefer signal over filler but epistemic hedges are signal; (2) **medium identification** — describe the rendering medium in plain, observable vocabulary that matches the specific surface behavior visible in the image; no menu, no checklist, no letter bucket; the observation drives the vocabulary, and technique words must be self-consistent with the surface just named (no mixing vocabulary from incompatible media in the same image); (3) **`[Art Style]` is the STYLE CANON** — the captioner must copy the meta-prompt's `style_foundation` content into `[Art Style]` verbatim (or near-verbatim) so every caption carries the same shared style DNA forward; the captioner's per-image writing freedom lives entirely in the observation blocks (`[Subject]`, `[Color Palette]`, `[Composition]`, `[Lighting & Atmosphere]`); rewriting the canon in the captioner's own voice is forbidden; (4) the **observations-vs-rules rule** — specific body parts, named objects, proper nouns, actual colors, pose details belong in the observation blocks, NEVER in `[Art Style]`; (5) the **5-slot canon skeleton** — How to Draw / Shading & Light / Color Principle / Surface & Texture / Style Invariants — is the expected structure of the canon the captioner preserves; anti-name preamble forbids genre labels (`3D CGI of X`, `cel-shaded anime`, `{Artist}-style`); character proportions live in `[Subject]`, not here; target 400-800 words; (6) **output-format** discipline — labels are plain `[Section]`, never `**[Section]**` — plus the unconditional **forbidden-terms** list (`cartoon`, `stylised`, `beautiful`, `epic`, `cinematic`, …). The meta-prompt itself arrives as section-delimited markdown from `PromptTemplate.render()` (see root CLAUDE.md). `thinking_level` defaults to MINIMAL (preserves prior behavior) and is threaded from `Config.caption_thinking_level` via `CaptioningService`. Zero-step `CAPTION_PROMPT` uses 5 fixed sections (Art Style / Subject / Color Palette / Composition / Lighting & Atmosphere) with `[Subject]` as most important and a 1500-4000 word total target — at zero-step there's no canon yet, so the captioner writes its own generic rules into `[Art Style]` as raw material for the analyzer.
- Caption quality is validated after Gemini returns: (1) total-length floor via `_minimum_caption_chars` (dynamic: `max(600, target * 2)` chars); (2) per-anchor section floor — `[Art Style]` < 100 words or `[Subject]` < 80 words raises RuntimeError. Floors live in `caption.py::_ANCHOR_SECTION_MIN_WORDS` and are shared by `_validate_expanded_template`'s self-consistency check.
- `caption_sections.py`: `parse_labeled_sections` (single-pass `[Section]` dict parse, used by compliance and style-consistency checks) + `build_generation_prompt` (reorders caption so `[Subject]` leads the generator prompt, falling back to raw caption when either anchor is missing). `build_generation_prompt` accepts an optional `style_canon` argument — when the caption's `[Art Style]` block is missing or below the 100-word floor, the canon is injected directly so the generator always sees the canonical style assertions. This subject-first rendering is the key to reliable subject fidelity from text-to-image generation.
- `generate.py`: Gemini Flash image generation with semaphore + retry + disk cache. `generate_single` skips the API call if `output_path` already exists with >0 bytes — crash+resume skips the entire generation phase. Atomic temp-file + rename writes prevent partial files on crash. `thinking_level` defaults to MINIMAL (preserves prior behavior) and is threaded from `Config.generation_thinking_level` via `GenerationService`. `_GENERATION_SYSTEM` also instructs the generator to honor explicit proportion ratios (e.g. "3.2 heads tall") rather than drifting toward default realistic anatomy.
- `experiment.py`: `_caption_and_generate` returns 3-tuple `(captions, generated_paths, pairs)`. Failed images are dropped from the surviving lists; counts are derived by the caller from `len(fixed_refs)` and `len(generated_paths)`. `run_experiment` rejects experiments below `_MIN_COMPLETION_RATE` (50%) generation success. Caption+generation runs as a per-image pipeline — each image's caption→generate is a single chained task so generation starts as soon as each individual caption completes (no serial boundary between phases). Failures are per-image via `return_exceptions=True`. `replicate_experiment` runs N caption+generate+eval cycles per template for rigorous mode, computing per-image median scores (captions cache-hit, generation is stochastic).
- `analyze.py`: Zero-step parallel Gemini vision + reasoning-model style analysis → `StyleProfile` + initial `PromptTemplate` seeds. Bootstrap captions use a generic section format; the compilation prompt explicitly notes this so the template defines its own section names. Compilation prompt includes a watercolor-landscape JSON example for structural reference without content anchoring.

## evaluate & score

- Evaluation metrics must receive inputs matching their documented semantics — DreamSim compares against the specific paired reference (not a set centroid), HPS scores against the per-image caption (the generation prompt, not the meta-prompt).
- `evaluate.py`: Per-image paired metrics dispatched via `asyncio.to_thread`, Gemini/xAI vision comparison, and caption compliance. `_VISION_SYSTEM` includes calibration examples for MATCH/PARTIAL/MISS across style (2D + CGI), subject (including proportions-split), and composition, plus calibration examples for `style_gap` observations; a medium mismatch — described in plain observable vocabulary — is an explicit MISS on style regardless of palette. `_VISION_SINGLE_PROMPT` emits 5 verdict tags — `<style>`, `<subject>`, `<composition>`, `<medium>`, `<proportions>` — plus a `<style_gap>` block carrying 2-4 sentences of canon-actionable deltas between reference and generated. All verdicts are parsed by `_parse_vision_verdicts` into `VisionScores` and threaded to `MetricScores.vision_medium` / `vision_proportions` / `AggregatedMetrics.vision_medium*` / `vision_proportions*`. The `style_gap` string rides on `VisionScores.style_gap` → `MetricScores.style_gap` via `_merge_vision`, then gets aggregated per-experiment into `AggregatedMetrics.style_gap_notes` (tuple of deduplicated observations) via `knowledge.aggregate_style_gap_notes`. `STYLE_GAP_PATTERN` is the canonical parser. `_PAIRWISE_COMPARE_PROMPT` requests per-aspect verdicts (style, color, subject, composition) alongside overall winner; `_parse_pairwise_response` (module-level) extracts winner + rationale for both provider branches. `VERDICT_PATTERN` is the canonical regex consumed by `reporting/render.py`. `compute_caption_compliance` parses each caption once and returns `(CaptionComplianceStats, prose)` — `check_caption_compliance` and `compute_caption_compliance_stats` are thin wrappers. `compute_style_consistency` uses `parse_labeled_sections` for the `[Art Style]` block. `extract_style_canon(meta_prompt)` extracts the `## style_foundation` block body from a rendered meta-prompt, feeding the two directional canon metrics.
- `evaluate_images` returns `(scores, n_eval_failed)`. It returns zero-score sentinel `MetricScores` (all fields 0.0, `is_fallback=True`) for failed evaluations to preserve index alignment — the returned list always has the same length as input. `n_eval_failed` counts substitutions. Callers compute their own `completion_rate` and pass it to `aggregate()`. `aggregate()` filters out `is_fallback=True` scores before computing mean/std so sentinels don't contaminate aggregated metrics.
- Vision comparison is per-image (one Gemini call per image pair) with ternary verdicts (MATCH/PARTIAL/MISS → 1.0/0.5/0.0); failures degrade to PARTIAL (0.5) neutral defaults.
- Caption compliance checking verifies keyword coverage (meta-prompt section topics), labeled section marker presence (`[Section Name]` in caption text), section ordering (markers appear in expected order), section length balance (no single section dominates >50% of words), subject specificity (when `[Subject]` is present: min word count, facet coverage via `_SUBJECT_FACET_KEYWORDS`, and generic-vs-specific-word ratio), and two directional canon compliance signals: **style-canon fidelity** (longest-common-substring ratio between the caption's `[Art Style]` block and the meta-prompt's `style_foundation` canon, char-level via `difflib.SequenceMatcher` with whitespace-normalization on both sides, score = LCS / len(canon) — HIGH = good, verbatim copy scores ≈1.0, paraphrase breaks character-level contiguity and scores <0.2, scrambled-vocabulary captures can't fake a high score; implemented by `compute_canon_fidelity`) and **observation-boilerplate purity** (token-trigram overlap between the caption's per-image observation blocks `[Subject]`/`[Color Palette]`/`[Composition]`/`[Lighting & Atmosphere]` and `style_foundation` — LOW overlap = good, captioner observed rather than pasting canon into observation blocks; implemented by `compute_observation_boilerplate_purity`). Both return 1.0 for short/missing blocks or empty canon (no penalty without signal — canon fidelity's floors are `_CANON_FIDELITY_MIN_CANON_CHARS=200` and `_CANON_FIDELITY_MIN_BLOCK_CHARS=50`). All seven components flow through `compliance_components_mean` in `types.py` with a divisor derived from `len(fields(CaptionComplianceStats))`. `compute_caption_compliance` parses each caption once (via `parse_labeled_sections`) and returns `(stats, prose)` in a single pass so callers don't double-scan; it accepts an optional `meta_prompt` kwarg and internally calls `extract_style_canon` to drive the two directional metrics (callers that don't pass one get neutral 1.0 defaults).
- `scoring.py`: `composite_score`, `adaptive_composite_score`, `per_image_composite`, `classify_hypothesis`, `paired_promotion_test` (Wilcoxon + bootstrap CI), plus shared `compliance_mean(m)` and `metric_deltas(metrics, baseline)` helpers (consumed by `prompt/synthesis.py` and `prompt/review.py`). Weight constants + penalty constants live at module top. `_VISION_SUBJECT_FLOOR=0.35` / `_W_VISION_SUBJECT_FLOOR_PENALTY=0.05` form a floor-penalty on the subject dimension.
- `composite_score` includes a consistency penalty (0.30 weight) based on per-image std of DreamSim and color histogram, plus a completion penalty (0.15 weight) based on `completion_rate` — experiments with high variance or missing images are penalized. The final score is floor-clamped to 0.0 (never negative) to prevent `adaptive_composite_score` min-max normalization from producing nonsensical rankings.
- All metrics in `composite_score` are normalized to [0, 1] before weighting — HPS via `_normalize_hps` (ceiling 0.35), aesthetics /10, vision /10. Weights are shared constants (`_W_DREAMSIM`, `_W_HPS`, etc.) in `scoring.py` — both `composite_score` (aggregated) and `per_image_composite` (single image) derive from them.
- `per_image_composite()` applies the same weights as `composite_score` but to a single `MetricScores` (no variance penalty). Used for paired statistical testing.
- `paired_promotion_test()` uses Wilcoxon signed-rank (scipy.stats.wilcoxon, one-sided) with bootstrap 95% CI (2000 resamples). Falls back to sign test on ties. Threshold: p < 0.10 AND effect_size > 0.
- Hypothesis classification uses keyword matching in `classify_hypothesis()` against `CATEGORY_SYNONYMS` (defined in `taxonomy.py`, re-exported from `utils.py`, re-imported as `_CATEGORY_SYNONYMS` inside `scoring.py`) — extend the synonym map when adding new categories. `get_category_names()` in `types.py` unions the synonym-map keys into the ranked category pool so `suggest_target_categories()` (in `prompt/_format.py`) can surface unexplored synonym categories (`lighting`, `texture`, `background`, `caption_structure`, …) at priority 1.0 even in fresh runs.

## knowledge & state

- `knowledge.py`: Open problems in KB are merged across experiments (deduplicated by text, capped at 10), not replaced — earlier experiments' problems survive. `[HIGH]/[MED]/[LOW]` prefix stripped at creation time via `strip_priority_prefix` (shared with `reporting/render.py`); parsed priority overrides the code heuristic. Near-duplicate problems are merged via token Jaccard similarity (threshold 0.6). **Style-gap observations**: `aggregate_style_gap_notes(raw_gaps)` deduplicates per-experiment `<style_gap>` strings via the same Jaccard-threshold dedup (cap `_STYLE_GAP_NOTE_MAX = 8`); `append_kb_style_gap_observations(kb, new_notes)` appends them onto `KnowledgeBase.style_gap_observations` (bounded ring buffer, `_STYLE_GAP_KB_MAX = 20`) skipping near-duplicates of prior entries. The iteration-persistence phase calls `append_kb_style_gap_observations` on the selected (or top-ranked) experiment's notes so the reasoner's next-iteration context carries the frontier's canon-edit feedstock.
- Caption diffs compare iteration N-1's best captions against N-2's via `state.prev_best_captions` (stored before overwriting `last_iteration_results`). `build_caption_diffs` in `knowledge.py` accepts `prev_captions: list[Caption]` directly.
- `state.py`: Public JSON persistence. Transient fields (`review_feedback`, `pairwise_feedback`) are excluded from serialization — cleared each iteration.
- `state_codec.py`: Low-level encoders/decoders: `_Encoder` (handles Path, dataclasses, enums), plus `*_from_dict` helpers used by `analyze.py` for the style cache.
- `state_migrations.py`: Schema version constants (`_SCHEMA_VERSION=5`, manifest=3, iteration-log=1, promotion-log=1) and payload migrators. v3→v4 backfills direction/mechanism fields on `experiment_history` and `last_iteration_results` (`direction_id`, `risk_level`, `failure_mechanism`, `intervention_type`, `changed_sections`); v4→v5 backfills neutral defaults for the medium-class + proportions vision dims on per-image and aggregated metric payloads; the `dino_similarity` → `dreamsim_similarity` rename is also handled here.
- `models.py`: `ModelRegistry` lazy-loads DreamSim / HPS / Aesthetics / SSIM with per-model locks; auto-detects MPS / CUDA / CPU.

## providers & retry

- `reasoning_client.py`: Provider-agnostic `ReasoningClient` supporting Anthropic / OpenAI / xAI / Z.AI / local (OpenAI-compatible) backends. Applies lightweight provider-specific prompt adaptations via `_adapt_prompts_for_provider` (GPT gets a user-message reminder for critical rules; GLM gets JSON-formatting reinforcement; others pass through unchanged).
- `ReasoningClient.call` and `.call_json` accept optional `temperature`, `reasoning_effort`, and `stage` kwargs. Effort resolution: the call-site value wins; `None` falls back to the client-level `default_reasoning_effort` (constructed from `Config.reasoning_effort`, exposed via `--reasoning-effort`, default `"medium"`). Call sites no longer hard-code `reasoning_effort="medium"` — the CLI flag is the single source of truth. Temperature varies per stage (brainstorm 0.9, rank 0.1, expand 0.3, synthesis 0.3, review 0.2, zero-step 0.3) to preserve sampling diversity on providers that honor it. Provider mapping: Anthropic — `"low"` → thinking disabled, `"medium"` → adaptive, `"high"` → enabled with `budget_tokens=_ANTHROPIC_HIGH_BUDGET_TOKENS` (16k, guarded `<` max_tokens); temperature only honored when thinking disabled (otherwise `_warn_silent_drop`). OpenAI — threads `reasoning.effort`, drops `temperature` with one-time warning. Z.AI/xAI/local — drop `reasoning_effort` with one-time warning per provider.
- Truncation detection: every `_call_*` method inspects provider-specific finish signals (Anthropic `stop_reason=="max_tokens"`, OpenAI `status=="incomplete"` + `incomplete_details.reason=="max_output_tokens"`, chat-completions `finish_reason=="length"`). On hit raises `TruncationError(provider, stage, max_tokens, completion_tokens)`. Gemini captioner raises the same when a candidate's `finish_reason` is `MAX_TOKENS`. `retry.async_retry` detects `TruncationError` by class name and re-raises immediately instead of burning retries on a deterministic failure.
- Observability: every provider call emits exactly one `API_CALL provider=… model=… stage=… duration_s=… [max_tokens=…] [effort=…] [thinking_level=…] [usage=…] status=…` log line via `retry.log_api_call` (`status` is one of `ok|error|truncated`). Stages are threaded from call sites: `analyze`/`compile` (analyze.py), `init_brainstorm`/`init_rank`/`init_expand` (prompt/initial.py), `brainstorm`/`rank`/`expand` (prompt/experiments.py), `synthesis`, `review`, plus `caption`/`generate`/`vision_single`/`pairwise` for Gemini surfaces.
- `retry.py`: `async_retry` with jittered exponential backoff (`delay * (0.5 + random())`), 429/ResourceExhausted detection with 30s base delay, per-request `asyncio.wait_for` timeouts scaled with the output-token budget via `gemini_timeout_s(max_output_tokens)` (60s floor for small vision verdicts, ≈670s for 32k-token captions; image generation uses a flat 300s), and per-surface circuit breakers: `caption_circuit_breaker`, `generation_circuit_breaker`, `vision_circuit_breaker` (60s pause after 15 consecutive failures). Anthropic streaming retries transient errors by exception class (`httpx.RemoteProtocolError`, `httpcore.ReadError`, `ConnectionResetError`, `BrokenPipeError`) first, with string-based fallback for unrecognized transient errors. `TruncationError` is never retried.
