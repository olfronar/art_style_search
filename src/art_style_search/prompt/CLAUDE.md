# prompt/ — reasoning-model interface

## Module map

- `__init__.py` - Public API re-exporting every symbol (`propose_experiments`, `propose_initial_templates`, `synthesize_templates`, `review_iteration`, `validate_template`, `enforce_hypothesis_diversity`, format helpers, etc.).
- `_format.py` - Dataclass-to-text rendering: `_format_style_profile`, `_format_template`, `_format_metrics`, `format_knowledge_base`, `suggest_target_categories` (PE2-inspired category-gap ranker).
- `_parse.py` - Holds `validate_template` (structural invariants only); JSON payloads flow through `prompt/json_contracts.py`. Required-anchor validation is table-driven via `_REQUIRED_SECTION_ANCHORS` / `_REQUIRED_CAPTION_ANCHORS` + `_check_anchors` helper (adding a third required anchor is a one-line edit). `_check_anchor_sub_blocks` additionally enforces mandatory sub-blocks inside the two anchor sections' `value` strings: `style_foundation` must contain `How to Draw:` (slot 1 of the six-slot [Art Style] skeleton — the other five slots are soft-guided via `CAPTION_SYSTEM` + all seeders, not validator-enforced); `subject_anchor` must contain `Proportions:` + at least one archetype token from `_SUBJECT_ANCHOR_ARCHETYPE_TOKENS`.
- `json_contracts.py` - JSON schema hints and `validate_*_payload` helpers per reasoner exchange type.
- `initial.py` - Zero-step template generation via brainstorm→rank→expand (mirrors the iteration flow in `prompt/experiments.py`). `brainstorm_initial_sketches` (1 call, `max_tokens=40000`) produces `num_branches*2` short `InitialTemplateSketch` ideas; `rank_initial_sketches` (1 call, `max_tokens=1000`) reorders them with a skeptical-reviewer prompt; `expand_initial_sketches` (N parallel calls, `max_tokens=24000`) turns each top-N sketch into a full 2000-8000 word template. `propose_initial_templates` orchestrates all three and pads with empty placeholders for failed expansions so `_sanitize_initial_templates` can swap in the compiled fallback.
- `experiments.py` - Per-iteration `propose_experiments` (single JSON batch of 8-12 raw proposals grouped into 3 directions) + `enforce_hypothesis_diversity` (drops duplicates keyed on `(category, failure_mechanism, intervention_type)`) + `select_experiment_portfolio` (picks 1 targeted per direction in D1→D2→D3 order, then fills remaining slots with bolds up to `num_branches`). System prompt uses compact non-negotiable rules + conditional context (full pipeline/metrics on first iteration, one-line reminder thereafter) + iteration strategy + diagnostic tips. Brainstorm includes `_BRAINSTORM_EXAMPLE` (good vs bad sketch). Post-expand `_validate_expanded_template` checks anchor positions, word count, changed_sections consistency. `_build_shared_proposal_user` adaptively skips Style Profile and XML template format after iteration 1; raises vision feedback limit to 500 when `plateau_counter > 3`. All new parameters (`is_first_iteration`, `iteration`, `plateau_counter`) default to preserve prior behavior.
- `synthesis.py` - `synthesize_templates`: merges top 2-3 experiments' best sections into a single template. Accepts `baseline_metrics` parameter; `_metric_strength_annotations` computes per-experiment metric deltas and annotates which sections are likely strong. Decision guidance maps metric improvements to section quality (e.g., color_histogram → color_palette section). Review includes `_REVIEW_EXAMPLE` calibration standards.
- `review.py` - CycleResearcher-inspired `review_iteration` (skeptical reviewer → `strategic_guidance`). Phased into assess → synthesize → advise. Includes calibration standards (SIGNAL requires 2+ metrics, <0.01 is noise, no grading on a curve). `_REVIEW_EXAMPLE` shows good vs bad assessments in user message.

## Conventions

- Post-expand template self-consistency: `_validate_expanded_template` rejects templates where `caption_length_target` is below the minimum plausible total implied by the declared `caption_sections` (2 anchors × 100w + ancillary × 80w, via `_minimum_plausible_caption_total`). This catches reasoners that ask for a 700w total caption while also declaring 10 sections — each would need ≥80w and the captioner always overshoots.
- Experiment section-count rules depend on `risk_level`: targeted experiments (`risk_level="targeted"`, the default) change exactly 1 section for clean attribution; bold experiments (`risk_level="bold"`) may change up to 3 related sections for broader mechanism tests. Each proposal declares both `changed_section` (singular, primary) and `changed_sections` (list) in the JSON contract — `_normalize_changed_sections` in `prompt/json_contracts.py` backfills whichever is missing and enforces `changed_sections[0] == changed_section`. `validate_template` in `prompt/_parse.py` enforces the per-risk-level count bounds.
- `propose_experiments` system prompt includes search principles: **Momentum** (double down on confirmed KB insights), **Boldness** (assume incumbent is conceptually wrong in at least one way), **Diversity** (no two proposals may share `(category, failure_mechanism, intervention_type)`). The user message includes the current composite score. Ranking prompt prioritizes by: (1) evidence fit, (2) risk-reward balance, (3) novelty.
- Hypothesis variability is enforced at 5 layers: (1) **Prompt-level** — the system prompt asks for 3 mechanism-tagged directions, and each proposal declares `target_category`, `failure_mechanism`, and `intervention_type`; multiple directions may touch the same category as long as mechanisms or interventions differ. The system prompt also carries a **structural exploration quota** requiring ≥1 proposal per batch to change either `caption_sections` (set or order), `caption_length_target` (≥30% delta), or the `section_names` set — pure value-rewording batches are called out as the most common failure mode; (2) **Post-parse dedup** — `enforce_hypothesis_diversity()` in `prompt/experiments.py` keys on the triple `(target_category, failure_mechanism, intervention_type)` (falling back to `classify_hypothesis()` keyword matching when `target_category` is empty) and drops duplicates; (3) **KB-guided targeting** — `suggest_target_categories()` (in `prompt/_format.py`) ranks categories by improvement potential (unexplored=1.0, partial success=0.7, diminishing returns=0.1) and injects the ranked list into the user message; (4) **Portfolio selection** — `select_experiment_portfolio()` in `prompt/experiments.py` trims the raw batch to `num_branches` by taking one targeted proposal per direction (preserving D1→D2→D3 priority order) and then filling remaining slots with bolds; (5) **Structural novelty swap** — when `incumbent_template` is passed, if every selected proposal shares the incumbent's structural signature (`template_structural_signature`: `(section_names, caption_sections, caption_length_target)`), the last bold is swapped for a structurally-novel candidate when one exists. `_is_structurally_novel` treats a `caption_length_target` delta of ≥30% as a novel axis. Both the dedup filter and the portfolio selector are called in `workflow/iteration_proposals.py` after `propose_experiments()` returns; structural entropy (distinct signatures / portfolio size) is logged per iteration.
- `ConvergenceReason.REASONING_STOP` (the reasoning model emitted `[CONVERGED]` or the parser returned zero proposals) is gated behind `_should_honor_stop()` in `workflow/policy.py` (see workflow/CLAUDE.md). The softened `[CONVERGED]` instruction in `prompt/experiments.py` also pushes the reasoning model to self-check the same conditions before emitting the token.
- Synthesis always runs when >= 2 experiments exist — top 2-3 by `adaptive_composite_score` are merged regardless of whether they individually beat baseline. This allows cherry-picking best sections from experiments that failed overall but improved different aspects. Synthesis reasoning runs in parallel with pairwise comparison and independent review; only the synthesis experiment (caption+generate+eval) runs after.
- The worst discarded experiment's details (hypothesis, caption, vision feedback) are shown to Claude for negative learning — helps avoid repeating failures.
- Data fed to the reasoning model across `propose_experiments` / `synthesize_templates` / `review_iteration` must appear via exactly one path — if the history formatter includes a field, don't also add a dedicated section for it (or vice versa).
